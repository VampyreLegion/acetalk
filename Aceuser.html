🧠 Executive Summary

AI music generator built on a Large Language Model + Diffusion Transformer (LLM+DiT) architecture. Unlike traditional DAWs, it does not use hard-coded parameters, MIDI routing, or virtual knobs. Instead, it interprets your input as semantic tokens.

Control is achieved through a Dual-Prompting System:

  • Tags Field: Defines the global sonic palette, production era, and performance techniques.
  • Lyrics Field: Controls timeline structure, instrument triggers, and dynamic shifts using bracketed syntax.

📐 Chapter 1: The Dual-Prompting Philosophy

DiT diffusion process treats your prompt as a spatial-temporal map. The model cross-references global descriptors (Tags) with time-anchored cues (Lyrics Brackets) to generate coherent arrangements.

ComponentPurposeBest Practice
TagsSets instrumental foundation, mood, era, production styleComma-separated, 5–12 tokens max. Order matters: place primary instruments first.
Lyrics/StructureMaps sections, triggers solos/drops, controls vocal behaviorUse [Bracketed] cues on separate lines. Keep descriptors concise. Never nest brackets.

⚠️ Critical Rule: Never mix structural brackets into the Tags field.
Tags = What it sounds like (Global)
Lyrics = When it happens (Temporal)

💡 Token Weighting & Solo Triggers

  • First 3 tokens carry the highest diffusion weight. Place your lead instrument and core mood first.
  • Add virtuosic or shredding to Tags if the model under-renders solos. These are high-weight tokens in the training set.
  • Avoid redundant terms (e.g., guitar, acoustic guitar, guitar riff → collapses into noise).

🗺️ Chapter 3: Lyric Brackets & Structural Control

Place structural and performance cues in the Lyrics field. Each bracket acts as a time-anchored trigger.

🏗️ Structural Brackets

TagEffect
[Intro]Low/medium energy. Establishes key, tempo, and primary loop.
[Verse]Narrative focus. Lead vocal/melody forward, lighter backing.
[Pre-Chorus]Rising tension. Risers, drum fills, harmonic shifts.
[Chorus]Peak energy. Full arrangement, loudest delivery.
[Bridge]Harmonic variation. Changes progression or slows pace.
[Outro]Fading energy. Strips instruments for a clean finish.
[Interlude]Instrumental connector. No vocals.

🎸 Dynamic & Instrumental Triggers

TagEffect
[Instrumental] / [Inst]Strips all vocals.
[Guitar Solo] / [Synth Solo]Directs the model to prioritize a specific instrument solo. (While [Solo: Instrument] is recognized, [Guitar Solo] yields more consistent results.)
[Arpeggio]Forces the synth or guitar into a repeating broken-chord pattern.
[Breakdown]Strips drums and heavy bass. Leaves a "clean" instrument section.
[Drop]Triggers high-intensity impact + heavy bass. Best for EDM/Metal.
[Build-up]Gradual volume/density increase. Rushing drums.
[Strumming]Switches string instruments to rhythmic chords.
[Riff]Repetitive, catchy instrumental hook.
[Fill]Short drum/synth flourish at phrase ends.

🎵 LRC Synchronized Lyrics

When your Lyrics field contains actual sung lyrics (not just structural brackets), a 🎵 LRC button appears on the job card after generation. Clicking it generates a time-synchronized .lrc file — the standard format used by music players (Poweramp, foobar2000, VLC) to display lyrics scrolling in sync with the audio. Timing is estimated from RMS silence-boundary detection in the audio; BPM is used as a fallback for subdivision. See the 🎵 LRC guide section for full details.

🔄 The "Repaint" Workflow

  1. Highlight the time range (e.g., 0:15–0:30).
  2. Adjust the Tags for that segment (e.g., change strummed guitar → finger-picked guitar).
  3. Click Repaint. The DiT model regenerates only that window while preserving surrounding diffusion coherence.

Multi-Mask Repaint

Available via the Extract button when multiple job cards are selected, or directly in the Cover tab's repaint workflow. Lets you repaint up to 3 non-overlapping time regions in a single generation pass — each region can have different tags. Uses the NyxAudioOverlay ComfyUI node to stitch results back into the original audio.

Stems Tab — Demucs Separation

Uses Facebook's Demucs v4 to separate audio into 4 stems (vocals, drums, bass, other) or 6 stems (adds guitar and piano). Deterministic — same input always produces the same output. Best for standard stem separation; faster and more predictable than Extract Mode.

ModelStemsBest For
htdemucs4 (vocals, drums, bass, other)General-purpose; fast; best starting point
htdemucs_6s6 (adds guitar, piano)When guitar or piano isolation is needed
mdx_extra4Alternative quality profile; sometimes cleaner on certain genres

📻 Radio Mode

Generates an endless, coherent audio stream. Each segment (~30 s recommended) inherits timbre from the previous via ReferenceTimbreAudio — the stream stays sonically consistent while evolving through random seeds. New segments queue in the browser and play back-to-back automatically.

  • Set XL Turbo, 8 steps for near-real-time streaming (segment generation ≈ segment duration)
  • History panel shows all completed segments with download links; 🗑 Clear removes the display without stopping the stream
  • Refreshing the page reconnects to the running stream automatically

Recommended Iterative Workflow

  1. Analyze a reference track → Apply BPM/Key/Chords
  2. Generate at 30 s, XL Turbo (fast preview)
  3. Retake if it's fundamentally wrong; Repaint specific sections if most is right
  4. Extend to build to full length
  5. Variation batch (×4) to audition different performances at full length
  6. Cover tab with SFT model for a high-quality render of the best variation
  7. Demucs to separate stems; use stems in your DAW
  8. LRC if you added lyrics — download .lrc alongside the audio

🛑 Chapter 8: Troubleshooting & Pro Checklist

IssueSolution
Solo ignores instrumentAdd virtuosic or shredding to Tags. Ensure instrument is in first 3 tags. Use [Guitar Solo] in Lyrics.
Vocals drown out instrumentsAdd dry mix, forward vocal, spacious arrangement to Tags.
Drop feels weakPlace [Drop] on its own line. Add heavy bass impact, full arrangement to Tags.
Filter sweep doesn't triggerUse [Interlude - sweeping resonance, evolving texture] in Lyrics + filtered, sidechained in Tags.
Model ignores structureKeep brackets on separate lines. Avoid nesting.
Cover mode doesn't match reference styleRun Analyze tab first → Apply to set BPM/Key/Chords from the reference. Use XL SFT model at 30–50 steps, CFG 7. Turbo model at high CFG can over-generate away from the reference.
Extract mode output sounds like a full mix, not isolatedLower denoise to 0.7–0.85 for more reference preservation. Use very specific tags: isolated vocals, a cappella, no instruments, no drums — vague tags produce vague results. Use XL Base model.
Radio stream stops or never starts playbackCheck the Segment History panel — if segments appear there, audio is generating. If the player doesn't auto-play, click ▶ on a history entry manually. Ensure browser hasn't blocked autoplay (check browser console for NotAllowedError). If history is empty, check the MusicWeb service logs: journalctl -u musicweb -n 50
Radio segments don't sound coherent (too random)The timbre chain requires the previous segment file to exist in ComfyUI's output directory. If files are being deleted or moved, coherence breaks. Ensure disk is not full (df -h). Reduce Duration to 30 s to keep the chain shorter.
LRC button not showing on job cardThe 🎵 LRC button only appears when lyrics were provided at generation time. Check history — if the job's caption includes lyrics text, the button should appear. If missing, regenerate the track with lyrics in the field.
LRC timing is badly offTiming is estimated from audio silence-boundary detection. Works best when lyrics have distinct phrase-break silences (≥0.8 s). For continuous vocal tracks, set the correct BPM in the Parameters tab before generating LRC. For frame-accurate sync, import the .lrc into your DAW and adjust manually.
Quality score is low on Spectral dimensionBass-heavy: add bright, crisp, airy tags or reduce bass instrument density. Treble-heavy: add warm, full, deep bass. Also check if the output is clipping (Saturation score low) — regenerate or reduce output level.
Demucs separation has bleed (instruments leaking into wrong stem)This is normal for htdemucs on complex material. Try mdx_extra model as an alternative. For cleaner vocal isolation, the 6-stem model adds guitar/piano which reduces "other" stem bleed.
ComfyUI workflow error in logsMost workflow errors are IndexError: list index out of range on the negative conditioning input — this means the workflow template has an invalid node reference. All templates should use a ConditioningZeroOut node for negative conditioning. Check workflow_*_template.json files in AceUser/.

  • Brackets are in Lyrics field only, one per line, no nesting
  • Production descriptors match desired DAW/module behavior
  • Repaint/Extend strategy planned for iterative refinement
  • Seed saved for reproducibility if output is strong
  • For Cover/Extract: Analyze reference track first and Apply results

📎 Appendix & Interactive Prompt Builder

Run this Python script in your terminal to generate validated, optimized prompts with automatic syntax checking.

#!/usr/bin/env python3
"""
Prompt Architect v1.1
Interactive CLI tool to generate optimized Tags + Lyrics fields
"""

import sys

def print_header():
    print("\n" + "="*60)
    print(" 🎛️  PROMPT ARCHITECT (v1.5+)")
    print("="*60 + "\n")

def get_input(prompt, default=""):
    val = input(f"{prompt} [{default}]: ").strip()
    return val if val else default

def validate_tags(tags):
    warnings = []
    tokens = [t.strip() for t in tags.split(",") if t.strip()]
    
    if len(tokens) > 15:
        warnings.append("⚠️ Too many tags. DiT models perform best with 5-12 focused tokens.")
        
    if any("[" in t or "]" in t for t in tokens):
        warnings.append("⚠️ Brackets belong in the Lyrics field, not Tags.")
        
    if "solo" in tags.lower() and any("solo" in t.lower() for t in tokens):
        warnings.append("💡 Tip: Move [Solo: Instrument] or [Guitar Solo] to Lyrics. Keep only instrument names in Tags.")
        
    return warnings

def build_prompt():
    print_header()
    
    print("📌 STEP 1: GLOBAL STYLE TAGS")
    print("(Comma-separated. Lead instrument + mood + production + era)")
    tags = get_input("Enter Tags")
    
    warnings = validate_tags(tags)
    for w in warnings:
        print(w)
        
    print("\n📌 STEP 2: SONG STRUCTURE")
    print("Enter structure line by line. Press Enter on empty line to finish.")
    print("Use: [Intro], [Verse], [Chorus], [Bridge], [Drop], [Breakdown], etc.")
    print("Use: (harmonies), UPPERCASE, [Section - Adj], [Vocal: Type]")
    
    lyrics_lines = []
    while True:
        line = input("Lyrics/Bracket: ").strip()
        if not line:
            break
        lyrics_lines.append(line)
        
    lyrics = "\n".join(lyrics_lines)
    
    print("\n" + "="*60)
    print(" ✅ GENERATED PROMPT")
    print("="*60)
    print("\n📤 TAGS FIELD:")
    print(tags)
    print("\n📝 LYRICS FIELD:")
    print(lyrics)
    print("="*60)
    print("\n💡 Usage Tips:")
    print("1. Paste Tags into the 'Style/Prompt' box.")
    print("2. Paste Lyrics into the 'Lyrics' box.")
    print("3. If a section needs fixing, use Repaint + adjust only that section's tags.")
    print("4. Add 'virtuosic' to Tags if solos lack detail.")
    print("="*60 + "\n")

if __name__ == "__main__":
    try:
        build_prompt()
    except KeyboardInterrupt:
        print("\n\n👋 Prompt generation cancelled.")
        sys.exit(0)
    except Exception as e:
        print(f"\n❌ Error: {e}")
        sys.exit(1)
