What kind of Python UI do you want?

This shapes the whole architecture — pick the approach that fits how you work

A

Web App (Flask + HTML/CSS/JS)

Runs locally in your browser. Rich UI with custom styling, tabs, panels. You already have HTML knowledge from the PDF scripts. Full dark-mode, slick design possible. No extra installs beyond Flask.

  • ✅ Best looking UI
  • ✅ Works in any browser
  • ✅ Easy to extend later
  • ⚠️ Slightly more code to wire up
B

Gradio App

Python-native ML UI framework — already used in ACE-Step's Gradio demo. Rapid to build, runs in browser. Great for dropdowns, text areas, tabs. Familiar if you've used ACE-Step standalone.

  • ✅ Very fast to build
  • ✅ ML community standard
  • ✅ Clean tabbed layout out of the box
  • ⚠️ Less visual flexibility than custom HTML
C

PyQt6 Desktop App

Native desktop window — no browser needed. Full control, offline, resizable panels. More traditional GUI feel with menus, toolbars, rich text editors.

  • ✅ True standalone desktop app
  • ✅ No browser dependency
  • ✅ Best for complex editors
  • ⚠️ More complex to build
  • ⚠️ Heavier install (Qt)
